## ◈ Core Features

<details>
<summary><strong>⬡ &nbsp; Technical Interview-as-a-Service</strong></summary>

<br/>

Every interview is a structured, 60-minute deep technical session conducted by a **domain-matched senior engineer**. Not a recruiter. Not an algorithm. A human expert who has built what they're evaluating.

- **Domain-matched interviewers** — aligned to the exact stack and seniority of the role
- **Standardized rubric** — consistent scoring across all candidates, reducing bias
- **Full session recording** — every question, every answer, preserved for review
- **Detailed feedback report** — structured observations delivered to HR within hours
- **One-click rebooking** — if a candidate advances or fails, the next step is one action away

</details>

---

<details>
<summary><strong>⬡ &nbsp; Async Coding Engine</strong></summary>

<br/>

Real-world engineering problems, not algorithm trivia. Candidates receive a time-bound challenge that mirrors the actual work of the role. Code executes in an **isolated Docker container** — language-aware, tamper-proof, and instantaneous.

- **Monaco / CodeMirror editor** — VSCode-grade in-browser coding environment
- **Docker sandbox execution** — containerized, isolated, reproducible
- **Multi-language support** — JavaScript, Python, Java, C++
- **Time-bound sessions** — 30 to 60 minutes, auto-terminated
- **Auto evaluation** — output comparison, execution trace, complexity analysis
- **Anti-cheating layer** — session integrity enforcement at the container level

</details>

---

<details>
<summary><strong>⬡ &nbsp; Skill Graph Matching Engine</strong></summary>

<br/>

Behind every interview booking is a precision matching system. Not keyword search. **Weighted graph traversal** across skills, experience depth, domain history, and real-time availability.

- **Skill-tag taxonomy** — structured, hierarchical skill mapping
- **Experience weight scoring** — seniority is quantified, not assumed
- **Availability filter** — real-time calendar synchronization
- **Rating-based ranking** — past interview performance influences future matching
- **Phase 2:** Neo4j-powered skill graph with ML-assisted recommendations

</details>

---

<details>
<summary><strong>⬡ &nbsp; Expert Interviewer Marketplace</strong></summary>

<br/>

HireFlow maintains a curated network of verified senior engineers — available on-demand as technical interviewers. Each interviewer is vetted, rated, and domain-classified.

- **Verified profile system** — background, stack, and experience validation
- **Accept / Decline workflows** — interviewers control their availability
- **Structured session briefings** — role context, rubric, and candidate profile delivered pre-interview
- **Earnings dashboard** — transparent per-interview compensation and payout history
- **Performance tracking** — rating accumulation influences matching priority

</details>

<details>
<summary><strong>⬡ &nbsp; Structured Rubric & Evaluation System</strong></summary>

## ◈ Design & Experience Manifesto

> *We didn't build a dashboard. We built a world.*

### The Visual Architecture of HireFlow

HireFlow's interface is engineered with the same rigor as the backend — every animation has a frame budget, every transition has semantic meaning, every interaction earns its existence.

**Parallax Depth System**  
The hero exists across five layered planes — background particles, mid-depth grid, floating nodes, foreground glassmorphic card, and the typography layer. As you scroll, each layer moves at a distinct velocity, creating genuine three-dimensional depth in a two-dimensional medium.

**3D Network Globe**  
Rendered with React Three Fiber and Three.js — a live, rotating network graph representing the global interviewer-candidate connection fabric. Nodes represent skill domains. Edges represent active matching paths. The camera follows cursor movement, giving the user a sense of control over something vast.

**Floating Skill Nodes**  
Across the landing and dashboard, skill nodes drift in ambient motion — never still, never distracting. Each node carries a tag, a glow, and a subtle physics response to proximity. They communicate abundance without noise.

**Scroll-Triggered Reveals**  
GSAP ScrollTrigger drives every section entrance — content doesn't appear, it *arrives*. Text splits and cascades. Cards lift from below. Counters spin to their values. Nothing is static because nothing in engineering is static.

**Micro-Interaction Language**  
Every interactive element speaks. Buttons ripple on press. Form fields breathe on focus. Cards elevate 4px with a subtle shadow expansion on hover. The system has a physical vocabulary that rewards attention.

**Custom Cursor Distortion**  
The cursor is not a pointer — it is a lens. A WebGL distortion layer follows the cursor, subtly bending the canvas beneath it, as if reality has slight weight. It is subtle. It is unforgettable.

**Glassmorphism Cards**  
Feature cards, pricing cards, and evaluation scorecards use multi-layer glassmorphism — frosted depth, soft luminance halos, and translucent borders. They feel like physical objects on a lit surface.

**Animated Counters**  
Achievement metrics don't sit static — they animate upward as the section scrolls into viewport. Each number counts from zero with an easing curve that decelerates into its final value.

**Smooth Page Transitions**  
React Spring orchestrates page-level transitions — views dissolve and reform, never jarring, always intentional. The app feels like a continuous space, not a collection of disconnected pages.

**Dark Premium Theme**  
No white mode. HireFlow lives in a calibrated dark spectrum — near-black backgrounds (`#0A0A0F`), electric accent (`#6C3CE1`), luminous highlights, and a typography system that glows rather than merely renders.

```

       │
       ▼

       │                                              │
       ▼                                              ▼
  Matching Engine Selects Interviewer        Auto Evaluation Result Captured
       │
       ▼

       │
       ▼

       │
       ▼
  Interviewer Submits Rubric Scorecard
       │
       ▼
  Scorecard + Recording Delivered to HR
       │
```

## ◈ Tech Stack

### Frontend

| Layer | Technology | Purpose |
|---|---|---|
| Core | React + Vite (TypeScript) | Application framework |
| 3D Rendering | Three.js + React Three Fiber | Globe, nodes, spatial UI |
| Animation | GSAP + ScrollTrigger | Timeline control, scroll animations |
| Motion | Framer Motion | Page transitions, entrance effects |
| Micro-interaction | React Spring | Physics-based interactions |

### Backend

| Layer | Technology | Purpose |
|---|---|---|
| Runtime | Node.js (TypeScript) | API server core |
| Framework | Fastify / Express | REST API routing |
| Auth | JWT + RBAC | Session and role management |
| Primary DB | PostgreSQL | Relational data, transactions |
| Cache | Redis | Session cache, fast lookup |

## ◈ Future Vision

> *This is version one. The vision goes much further.*

**AI-Based Hire Success Prediction**  
Every scorecard generated by HireFlow will feed a performance model. Over time, the system will predict the probability of hire success at 6-month and 12-month post-hire benchmarks — learning from outcomes, calibrating its own rubrics.

**Performance Analytics Loop**  
A closed-loop system where interviewer performance, candidate outcomes, and company retention data inform the matching algorithm continuously. The platform gets better with every interview conducted.

**Enterprise Analytics Dashboard**  
Global hiring teams need visibility across roles, geographies, and time. HireFlow's Phase 2 analytics layer will surface aggregate trends, bias detection signals, and hiring velocity metrics — customizable per org.

**Global Expansion**  
HireFlow begins in India, where the senior-interviewer deficit is most acute. The platform architecture is jurisdiction-agnostic — multi-currency, multi-timezone, and localization-ready for Southeast Asia, MENA, and Europe.
